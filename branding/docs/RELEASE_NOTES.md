## Tips for Release Notes

### Best Practices
1. **Lead with Benefits** - Focus on what users get, not just features
2. **Use Clear Language** - Avoid jargon; explain technical terms
3. **Be Exciting** - Show enthusiasm for the update
4. **Keep It Scannable** - Use bullet points and short paragraphs
5. **Include What's Next** - Set expectations for future updates

### Length Guidelines
- **App Store Short Version**: 200-400 characters
- **App Store Full Version**: 1,000-2,000 characters
- **In-App Release Notes**: 500-1,000 characters
- **Blog Post**: 800-1,500 words

### Key Messages to Include
- Rebranding significance
- AI-native UX benefits
- User impact
- What's improved
- What's coming next
